django 框架优化迁移数据库慢的问题

2018-05-19 15:18:50 | django

问题描述

使用django的方式管理数据库,主要操作为：增删改查表结构

生成迁移文件:
python manage.py makemigrations

执行迁库:
python manage.py migrate

久而久之,由于数据表结构越来越多,上述两部操作会越来越耗时,甚至达到2个小时.

解决办法

前提条件: 确定数据库中的表结构与当前最新的代码是一致的

  1. 删除项目中所有app目录下的migrations/xx 文件夹下的迁移文件,仅保留__init__.py
  2. 执行python manage.py makemigrations,重新生成所有迁移文件
  3. 删除数据库中django_migration表中的所有数据
  4. 执行python manage.py migrate --fake
