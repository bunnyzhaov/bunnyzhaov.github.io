比特币是如何做到去中心化的(上)

2018-02-11 15:39:05

中心化与去中心化的关系

去中心化是一个非常重要的概念，但是需要注意，去中心化并不是比特币独有的特性。

从历史发展的角度来讲，先有去中心化分享网络，再有比特币，所以不是因为有了比特币才有去中心化的概念，不过是比特币让去中心化这个词成为热门词汇。

常见的去中心化系统:

  • 互联网系统
  • 简单邮件传输协议SMTP

中心化与去中心化并非水火不容，其实没有任何一个系统是完全中心化或者完全去中心化的。

比如电子邮件系统是一个去中心化的系统，任何人只要愿意都可以实现用邮件与人沟通，但是现实却是大多数人使用者中心化的邮件服务，比如谷歌邮箱，QQ邮箱。

比特币系统是去中心化的系统，但是比特币交易所，钱包软件等却是中心化的。

P2P（peer-to-peer）网络技术发展

互联网发展初期，互联网是很简单的，而且接入速度非常慢，网上的资源也并没有像现在这么丰富，服务器也并不这么强悍，一台服务器承载几百人的访问也是绰绰有余。

此时的网站架构就可以理解为传统的C/S模式，由中心化的网站运营者或服务提供商假设服务器和带宽，为所有终端用户提供服务。

但是随着互联网的普及，互联网用户呈现爆炸式的增长，网站和数据服务商面临的压力越来越大，尽管网站和数据服务商想尽一切办法扩充服务器，增加网络带宽，但是仍然无法满足呈指数增长的互联网用户量。

密码学的哈希函数

2018-02-01 21:07:29

哈希函数是一个数学函数,具有三个特征:

  1. 输入可以是任何长度的字符串
  2. 它能产生固定大小的输出,如256位
  3. 它能进行有效计算。即对于特定的输入字符串,在合理时间内,可以计算出哈希函数的输出

要是哈希函数达到密码安全,还要求哈希函数具有以下三个附加特性：

  1. 碰撞阻力: 对于两个不同的输入,不会产生相同的输出
  2. 隐秘性: 根据哈希函数的输出,无法推算出函数的输入
  3. 谜题友好: 如果一个人要找到输出Y值所对应的输入X,假定在某输入集合中,那么它将非常难以求得X,只能用集合中的所有元素去一一尝试

碰撞阻力的应用

信息摘要：检验信息完整性

隐秘性的应用

承诺: 验证承诺的真实性

谜题友好

挖矿: 工作量证明机制的基础,难于求解但是非常易于验证

    python logging的组织模式

    1. logger 日志入口: 定义写入日志的内容和消息级别
    2. handler 处理方式: 定义如何处理logger传递来的日志记录, 比如是在控制台状态打印还是写入日志文件
    3. formatter 文本格式: 定义日志的文本组织方式
    4. filter 过滤器: 对logger传递给hanlder的消息进行额外的控制
